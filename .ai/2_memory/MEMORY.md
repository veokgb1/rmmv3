## 项目基本信息
- **项目名称**：RMM V3 (个人资金管理系统第三版)
- **开发者**：单人开发
- **立项时间**：2026年3月
- **目标用户**：个人用户，需要管理微信/支付宝/银行多渠道账单

## 核心痛点（为什么做 V3）
- V1/V2 已验证核心需求，但代码质量差、架构混乱，难以维护。
- V3 目标：用正确的架构重写，支持多账单来源、自动分类、数据可视化。

## 关键技术决策记录
| 日期 | 决策 | 原因 |
|------|------|------|
| 2026-03 | 使用 Firebase 而非自建后端 | 降低运维成本，快速验证 |
| 2026-03 | Vite + React + TypeScript | 类型安全，开发体验好 |
| 2026-03 | Tailwind CSS | 快速构建 UI，无需维护 CSS 文件 |
| 2026-03 | 禁止从 V2 复制代码 | V2 无类型注解，架构不清晰 |

## 约定俗成的命名规范
- 组件文件：PascalCase，如 `BillList.tsx`
- 工具函数：camelCase，如 `parseCsvFile.ts`
- 常量：SCREAMING_SNAKE_CASE，如 `MAX_FILE_SIZE`
- Firestore 集合名：小写复数，如 `transactions`, `categories`
