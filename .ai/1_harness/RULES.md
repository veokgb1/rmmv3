# RULES.md — 业务规则与判定逻辑
> 从 PRD 中提炼的核心业务规则，AI 必须严格遵守。

---

## 一、账单入账规则

### 1.1 支持的数据来源
| 来源 | 文件格式 | 识别关键字 |
|------|----------|------------|
| 微信支付 | CSV | `微信支付账单`, `交易时间`, `交易类型` |
| 支付宝 | CSV | `支付宝交易记录`, `交易创建时间`, `收/支` |
| 手动录入 | 表单 | 用户直接输入 |
| 银行账单 | CSV/XLSX | 待扩展 |

### 1.2 必填字段映射
每一条账单记录，无论来源，最终必须能映射到以下核心字段：
```
{
  date: string,        // 交易日期 YYYY-MM-DD
  amount: number,      // 金额（正数=收入，负数=支出）
  category: string,    // 分类（见分类体系）
  description: string, // 交易描述/备注
  source: string,      // 数据来源（wechat/alipay/manual）
  rawData: object      // 原始数据（完整保留，不丢失）
}
```

### 1.3 容错规则
- 金额字段含有货币符号（¥、$）→ 自动去除符号，保留数字。
- 日期格式不统一 → 尝试多种格式解析，失败则标记为 `PARSE_ERROR` 而非丢弃。
- 分类无法识别 → 归入「未分类」，不阻断入账流程。
- 重复交易检测 → 基于 (date + amount + description) 三元组判断，标记疑似重复而非自动删除。

## 二、分类体系

### 2.1 一级分类（固定，不可删除）
- 餐饮 / 交通 / 购物 / 娱乐 / 医疗 / 居住 / 教育 / 工资 / 副业 / 理财 / 转账 / 未分类

### 2.2 二级分类（用户可自定义）
- 每个一级分类下允许用户创建子分类。
- 子分类数据存储在 Firestore `categories` 集合中。

---

## 三、统计判定规则
- 「本月支出」= 当月所有 amount < 0 的记录之和（绝对值）。
- 「本月收入」= 当月所有 amount > 0 的记录之和。
- 「净收支」= 本月收入 - 本月支出。
- 转账记录（微信"转账"类型）：默认**不计入**支出统计，单独展示。

---

## 四、数据安全规则
- 所有用户数据存储在 Firebase，以 `userId` 作为数据隔离键。
- 未登录用户不得访问任何账单数据接口。
- 文件上传仅允许 .csv / .xlsx，大小限制 10MB。
