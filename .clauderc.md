## § 1 核心哲学

```
用户只管丢数据，系统负责兜底。
```

- 用户提交的数据可以是**混乱的、不完整的、格式不统一的**。
- 系统（AI + 代码逻辑）负责**清洗、解析、容错**，绝不把"格式不对"的问题甩回给用户。
- 数据处理优先级：`不丢失 > 正确性 > 优雅性`

## § 3 Git 提交规范（强制）

- Commit Message 必须使用**简体中文**
- 格式：`[类型] 简短描述 — 详细说明修改了什么/为什么`
- 合法类型：`[新增]` `[修复]` `[重构]` `[文档]` `[配置]` `[删除]` `[优化]`

**示例**：
```
[新增] 微信账单解析器 — 实现对微信CSV前16行元数据的跳过逻辑及字段映射
[修复] 金额解析错误 — 修复¥符号导致parseFloat返回NaN的问题
```

## § 5 AI 行为边界

1. **先读后写**：修改任何文件前必须先 Read，理解现有结构再动手
2. **最小侵入**：不得在未被要求时新增抽象层或"顺手优化"现有代码
3. **去重检查**：创建新文件前，检查是否已有同功能文件可扩展
4. **破坏性操作**：执行 `rm`、`reset --hard` 等操作前，必须先向用户确认
5. **输出风格**：简体中文，直接切入要点，避免废话前缀
