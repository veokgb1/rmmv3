# RMMV3 最高行为准则 (.clauderc.md)
> 本文件是 AI 助手在此项目中的最高行为约束。所有指令以此为最终依据。

---

## 核心哲学
**用户只管丢数据，系统负责兜底。**
- 用户提交的数据可以是混乱的、不完整的、格式不统一的。
- 系统（AI + 代码逻辑）负责清洗、解析、容错，绝不把"数据格式不对"的问题甩回给用户。
- 优先保证数据不丢失，其次保证数据正确性，最后再追求数据优雅性。

---

## 语言规范（强制执行）
- **所有代码注释**：必须使用【简体中文】。
- **所有 UI 文案**：必须使用【简体中文】。
- **所有文档说明**：必须使用【简体中文】。
- **每一行逻辑代码**：必须有中文注释，说明该行的意图。
- **严禁**：英文注释、拼音注释、无注释代码块。

---

## Git 提交规范（强制执行）
- Commit Message 必须使用**简体中文**。
- 格式：`[类型] 简短描述 - 详细说明修改了什么、为什么修改`
- 类型标签：`[新增]` / `[修复]` / `[重构]` / `[文档]` / `[配置]` / `[删除]`
- 示例：`[新增] 账单解析模块 - 实现对微信/支付宝账单CSV文件的自动识别与字段映射`

## AI 行为边界
1. 修改文件前必须先读取，理解现有结构再动手。
2. 不得在未被要求的情况下新增抽象层或"优化"现有架构。
3. 创建新文件时，检查是否已有同功能文件可复用。
4. 每次回复保持简洁，用简体中文，直接切入要点。
